# Deploy Merge Fruit API (free tier)

This guide deploys the Spring Boot API to **Render** (free web service) with **Neon** (free PostgreSQL). Both have generous free tiers suitable for a learning project.

| Service | Role | Free tier notes |
|---------|------|-----------------|
| [Neon](https://neon.tech) | PostgreSQL database | 0.5 GB storage, always available |
| [Render](https://render.com) | Runs the Dockerized API | Free web service; sleeps after ~15 min idle (cold start ~30–60 s) |

Your React game can stay on **Netlify** or **Vercel** (also free) and call the Render API URL.

---

## Overview

```
React (Netlify/Vercel)  →  Render (Spring Boot API)  →  Neon (PostgreSQL)
```

---

## Step 1 — Create a free PostgreSQL database (Neon)

1. Sign up at [neon.tech](https://neon.tech) (GitHub login works).
2. Create a project, e.g. `merge-fruit`.
3. On the dashboard, open **Connection details** and note:

   | Variable | Where to find it |
   |----------|------------------|
   | `DB_HOST` | Host (e.g. `ep-xxx.eu-central-1.aws.neon.tech`) |
   | `DB_NAME` | Database name (often `neondb`) |
   | `DB_USER` | Role name |
   | `DB_PASSWORD` | Password (shown once — save it) |

4. Keep **SSL** enabled (the prod profile uses `?sslmode=require` automatically).

You do **not** need to run `V1__init.sql` manually — **Flyway** applies it on first startup when `SPRING_PROFILES_ACTIVE=prod`.

---

## Step 2 — Deploy the API to Render

### Option A — Blueprint (recommended)

1. Push this repo to GitHub (already at `https://github.com/a1exymoroz/merge-fruit-api`).
2. Sign up at [render.com](https://render.com) and connect your GitHub account.
3. **New → Blueprint** → select the `merge-fruit-api` repo.
4. Render reads `render.yaml` and creates a web service.
5. In the service **Environment** tab, set the variables marked `sync: false`:

   | Key | Value |
   |-----|-------|
   | `DB_HOST` | from Neon |
   | `DB_NAME` | from Neon |
   | `DB_USER` | from Neon |
   | `DB_PASSWORD` | from Neon |
   | `CORS_ALLOWED_ORIGINS` | your frontend URL(s), comma-separated |

   Example CORS value if your game is on Netlify:

   ```
   https://your-game.netlify.app,http://localhost:5173
   ```

   `JWT_SECRET` and `ANONYMOUS_USER_PASSWORD` are auto-generated by the blueprint.

6. Click **Manual Deploy → Deploy latest commit** (or wait for auto-deploy on push).

7. When the deploy succeeds, your API base URL is:

   ```
   https://merge-fruit-api.onrender.com
   ```

   (Exact hostname appears on the Render dashboard.)

### Option B — Manual web service

1. **New → Web Service** → connect the GitHub repo.
2. Settings:
   - **Runtime:** Docker
   - **Plan:** Free
   - **Health check path:** `/api/scores`
3. Add environment variables (same table as above), plus:

   | Key | Value |
   |-----|-------|
   | `SPRING_PROFILES_ACTIVE` | `prod` |
   | `JWT_SECRET` | random string, at least 32 characters |
   | `ANONYMOUS_USER_PASSWORD` | any random internal string |

4. Deploy.

## Step 3 — Verify the deployment

Replace `YOUR_API_URL` with your Render URL.

```bash
# Leaderboard (public)
curl -s "https://YOUR_API_URL/api/scores?page=0&size=5"

# Sign up
curl -s -X POST "https://YOUR_API_URL/api/auth/signup" \
  -H 'Content-Type: application/json' \
  -d '{"email":"you@example.com","password":"YourPass123!","displayName":"Player1"}'

# Submit score (anonymous — no auth required)
curl -s -X POST "https://YOUR_API_URL/api/scores" \
  -H 'Content-Type: application/json' \
  -d '{"name":"Player1","score":4200}'
```

If signup returns a JSON error about the database, check Neon credentials and Render logs (**Logs** tab).

---

## Step 4 — Point your React game at the API

In your frontend `.env`:

```env
VITE_API_URL=https://YOUR_API_URL.onrender.com/api/scores
```

Redeploy the frontend on Netlify/Vercel after changing env vars.

## Cold starts (Render free tier)

Free Render services **spin down after ~15 minutes** without traffic. The first request after idle may take **30–60 seconds**. This is normal on the free plan.

To reduce cold starts without paying:

- Use a free uptime ping (e.g. [UptimeRobot](https://uptimerobot.com)) hitting `/api/scores` every 14 minutes — fine for demos, not ideal for production.

## Alternative free hosts

| Host | Pros | Cons |
|------|------|------|
| **Render + Neon** | Simple Docker deploy, good docs | Cold starts on free tier |
| **Railway** | Very easy UI | ~$5/month credit; may run out |
| **Fly.io** | Good Docker support | Requires credit card for verification |
| **Oracle Cloud Always Free** | Full VM, no sleep | More setup (Docker + systemd) |

The repo includes a `Dockerfile` that works on any of these platforms.

## Local Docker build (optional test)

```bash
docker build -t merge-fruit-api .

docker run --rm -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e DB_HOST=your-neon-host \
  -e DB_NAME=neondb \
  -e DB_USER=neondb_owner \
  -e DB_PASSWORD=your-password \
  -e JWT_SECRET=local-test-secret-at-least-32-characters-long \
  -e ANONYMOUS_USER_PASSWORD=internal-anon-password \
  -e CORS_ALLOWED_ORIGINS=http://localhost:5173 \
  merge-fruit-api
```

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `Required secrets are not configured` | Set `JWT_SECRET` (32+ chars), `DB_PASSWORD`, `ANONYMOUS_USER_PASSWORD` in Render env |
| DB connection timeout | Check `DB_HOST` / password; Neon project must be active |
| CORS error in browser | Add your frontend origin to `CORS_ALLOWED_ORIGINS` (include `https://`, no trailing slash) |
| 502 on first request | Free tier cold start — wait and retry |
| Flyway migration failed | Check Render logs; ensure Neon DB is empty or matches migration state |

---

## Security reminders

- Never commit `.env` — it is gitignored.
- Rotate `JWT_SECRET` if exposed; all existing tokens will invalidate.
- Use strong `DB_PASSWORD` from Neon; restrict DB access to Render outbound IPs only if Neon offers IP allowlisting on your plan.
